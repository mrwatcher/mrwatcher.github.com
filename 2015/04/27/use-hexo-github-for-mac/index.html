下面是我自己搭建的过程,仅供参考…

对于github pages的申请配置

  1. 对于github如何申请,如何跨过山的大海等等不做描述.教程很多!
  2. 根据要求创建好page这个项目后在项目目录下建立CNAME文件,在文件中添加一条你的域名,便可将DNS中设置好的域名利用别名跳转解析到xxx.io这个项目上了!

    对于hexo的安装和配置

    利用hexo可以将.md文件转换为pdf,html等利于阅读的现代文本格式,同理对于html的
    格式我们还可以将其上传到支持静态网页浏览的空间(SAE,GITHUB PAGE等),以组成有趣
    的个人blog。

    mac下hexo的安装

    hexo是一个免费的基于node.js的静态博客生成工具,所以先要在mac
    上装入nodejs.
    mac下安装nodejs可以使用>brew install nodejs命令进行安装或者直接从官网下载安装>curl -O http://npmjs.org/install.sh | sh亦可.
    安装完之后便可使用nodejs默认的包管理软件npm进行hexo的安装,执行>sudo npm install hexo -g进行安装,值得注意的是如果hexo未安装成功请尽量使用sudo命令获得权限进行安装,并且保证nodejs版本大于v0.10避免不必要的麻烦导致安装时失败.
    如果npm安装hexo发生未知错误时,在排除npm版本过低,nodejs版本过低,sudo权限等原因后,执行这两个命令>sudo npm config set user 0和sudo npm config set unsafe-perm true后再执行>sudo npm install hexo -g(以上只是针对于mac机…)

    hexo的配置

    将hexo安装完成后,便可使用>sudo hexo init 'path'在path目录下对hexo项目进行初始化,这一操作将生成其所需项目文件.
    顺利完成便可进行下一步操作了!
    如果已经安装git命令可以拉去一个相当漂亮的模板到thems目录下来增加颜值>git clone https://github.com/wuchong/jacman.git themes/jacman.
    在项目根目录下的_conf.yml便是整个项目的配置文件.可以在其中对必要参数,名称和thems进行修改.

后记:

  • markdown是一个非常爽的语法。非常适合写作,或者说就是为了写作而生的！
  • 值得注意的是hexo的默认配置文件(_config.yml),格式非常严格,需要在每项配置‘：’后面加入空格才能保证其格式正确.
  • 可以利用svn或者git开放局部source目录,来搭建团队技术blog.
  • md格式非常灵活,属于写作原格式.所以每次尽管已经生成了静态文件,对于原md文档,尽量保留备份才是明智选择！
